### ✅ 优势
1. **无需Python环境** - 双击即可运行

### ⚠️ 注意事项
1. **杀毒软件** - 可能被误报为病毒（添加信任即可）

3. **首次启动** - 第一次运行可能稍慢
4. **多进程** - 充分利用CPU多核心

## 🔧 方案2：自己打包EXE

如果你想自己打包（确保最新版本、安全性等）：

### 准备工作
1. **安装Python**
   - 下载：https://www.python.org/downloads/
   - 版本：Python 3.7 或更高
   - ⚠️ 安装时勾选"Add Python to PATH"

2. **下载源代码**
   - 下载整个项目文件夹
   - 确保包含以下文件：
     - `ultra_generator_windows.py`
     - `requirements.txt`
     - `build.bat`（或`build_exe.py`）

### 方法A：使用批处理脚本（最简单）⭐

**Windows用户推荐：**

1. 双击运行`build.bat`
2. 按提示操作
3. 等待打包完成（5-10分钟）
4. 在`dist`文件夹找到`BSC靓号生成器.exe`

```batch
# 完整流程
1. 双击 build.bat
2. 按任意键开始
3. 等待自动安装依赖和打包
4. 完成后自动打开 dist 文件夹
```

### 方法B：使用Python脚本

**macOS/Linux用户（交叉编译）：**

```bash
# 1. 安装依赖
pip3 install -r requirements.txt
pip3 install pyinstaller

# 2. 运行打包脚本
python3 build_exe.py

# 3. 等待完成
# 生成的EXE在 dist/ 目录
```

### 方法C：手动打包（高级用户）

```bash
# 1. 安装PyInstaller
pip install pyinstaller

# 2. 手动打包
pyinstaller --onefile \
  --name=BSC靓号生成器 \
  --console \
  --clean \
  --noconfirm \
  ultra_generator_windows.py

# 3. 查看结果
cd dist
# BSC靓号生成器.exe 已生成
```

## 🐛 常见问题

### Q1: 杀毒软件报毒？
**A**: 这是正常的误报
- PyInstaller打包的程序经常被误报
- 解决方案：
  1. 添加到杀毒软件信任列表
  2. 暂时关闭实时防护
  3. 自己打包（更放心）

### Q2: 双击无反应？
**A**: 可能原因：
1. 缺少运行库
   - 安装 Visual C++ Redistributable
   - 下载：https://aka.ms/vs/17/release/vc_redist.x64.exe
2. 被杀毒软件拦截
   - 检查杀毒软件日志
3. 权限问题
   - 右键 → 以管理员身份运行

### Q3: 速度很慢？
**A**: 检查：
1. CPU使用率是否100%（正常）
2. 进程数是否等于CPU核心数
3. 是否有其他程序占用CPU
4. 难度是否太高（8位以上很慢）

### Q4: 找不到生成的钱包？
**A**: 文件位置：
- 和EXE在同一目录
- 文件名：`ultra_vanity_wallets.txt`
- 如果找不到，搜索该文件名

### Q5: 可以在Mac/Linux上打包吗？
**A**: 可以，但需要额外配置：
- 方法1：使用Wine
- 方法2：使用虚拟机或Docker
- 方法3：使用CI/CD服务（GitHub Actions）

### Q6: 如何停止生成？
**A**: 按键盘快捷键：
- `Ctrl + C`（推荐）
- 或直接关闭窗口

### Q7: EXE文件太大？
**A**: 正常现象：
- 包含Python运行时：约15MB
- 包含所有依赖库：约15-20MB
- 总计：30-50MB
- 无法显著减小

### Q8: 可以改图标吗？
**A**: 可以：
```bash
# 修改打包命令
pyinstaller --onefile \
  --icon=your_icon.ico \  # 添加这行
  ultra_generator_windows.py
```
