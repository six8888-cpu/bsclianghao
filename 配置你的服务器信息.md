# ⚠️ 重要：配置你的服务器信息

## 📝 说明

为了安全起见，我已经移除了所有密码和服务器IP地址。

在使用部署脚本之前，你需要先配置服务器信息。

---

## 🔧 需要修改的文件

### 1. 一键部署_192核.sh
```bash
# 修改第4-6行
SERVER_IP="你的服务器IP"          # 改为：YOUR_SERVER_IP<==
SERVER_USER="root"
SERVER_PASSWORD="你的密码"        # 改为你的实际密码
```

### 2. 部署V2到服务器.sh
```bash
# 修改第10-14行
SERVER_192="你的服务器IP"         # 改为：YOUR_SERVER_IP<==
SERVER_8="你的服务器IP_2"         # 改为：YOUR_SERVER_IP<==
USER="root"
PASS_192="你的密码"              # 改为你的192核服务器密码
PASS_8="你的密码"                # 改为你的8核服务器密码
```

### 3. 更新命令.sh
```bash
# 修改第10、16行
echo "【192核服务器】你的服务器IP"    # 改为实际IP
echo "【8核服务器】你的服务器IP_2"    # 改为实际IP

# 修改第28、29行
echo "正在上传到 你的服务器IP..."
scp ... root@你的服务器IP:~/
```

## 🔒 安全建议

### 推荐方式：使用SSH密钥（不需要密码）

1. **生成SSH密钥**
   ```bash
   ssh-keygen -t rsa -b 4096
   ```

2. **复制公钥到服务器**
   ```bash
   ssh-copy-id root@YOUR_SERVER_IP<==
   ssh-copy-id root@YOUR_SERVER_IP<==
   ```

3. **修改脚本（不使用密码）**
   ```bash
   # 将所有 sshpass -p "$PASSWORD" 替换为空
   # 将所有 ssh 和 scp 命令直接使用，不需要密码
   ```

### 为什么要这样做？

✅ **使用SSH密钥的好处：**
- 更安全（不需要在脚本中存储密码）
- 更方便（不需要每次输入密码）
- 符合安全最佳实践

❌ **密码存储在脚本中的风险：**
- 容易泄露（推送到GitHub会暴露）
- 不符合安全规范
- 如果脚本被窃取，服务器会被入侵

## 💡 推荐：创建本地配置文件

创建一个本地配置文件（不提交到Git）：

```bash
# 创建 server_config.sh（已在.gitignore中）
cat > server_config.sh << 'EOF'
#!/bin/bash
# 服务器配置（本地使用，不提交到Git）

export SERVER_192="YOUR_SERVER_IP<=="
export SERVER_8="YOUR_SERVER_IP<=="
export PASSWORD_192="你的密码"
export PASSWORD_8="你的密码"
EOF

chmod +x server_config.sh
```

然后在其他脚本中引用：
```bash
# 在脚本开头
source ./server_config.sh
```
